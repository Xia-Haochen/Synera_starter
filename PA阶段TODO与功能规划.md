# PA阶段 TODO 与功能规划（按最优推进顺序）

说明：以下顺序按“先打基础数据结构 -> 再打战斗闭环 -> 再做运营系统 -> 最后扩展优化”的最稳妥路径排列。

## 阶段0：基础重构（先做，决定后续上限）

| TODO | 目标功能 | 产出 |
|---|---|---|
| 0-1 统一核心数据模型 | 明确 `Player/Unit/Board/Bench/RoundState` 的字段和关系 | 可扩展的数据结构 |
| 0-2 重构 Unit 体系 | `Unit` 基类 + 3~5 个派生英雄（多态技能接口） | 战斗逻辑可挂接 |
| 0-3 事件与更新主循环 | 准备/战斗/结算状态驱动器（tick 或定时器） | 稳定游戏主流程 |
